# Link-U 웹앱 고도화 백로그

기준일: 2026-04-25  
목표: Android 네이티브 NFC 쓰기 흐름과 웹 관리 흐름을 운영 관점에서 자연스럽게 연결한다.

---

## 1) 우선순위 A (즉시 착수 권장)

- [ ] **NFC 태그 관리 UX 통일**
  - 반려 상세/대시보드 홈/주체별 화면의 NFC 섹션 컴포넌트 정렬
  - 버튼/상태 문구/락 상태(`tenantSuspended`) 표현 통일

- [ ] **네이티브 handoff 가시성 강화**
  - 웹에서 `앱으로 기록` 진입 후 상태(대기/완료/실패) 피드백 강화
  - 재시도 경로와 오류 메시지 표준화

- [ ] **태그 이력 가독성 개선**
  - 최근 기록(성공/실패/원인) 시간순 노출
  - 운영자가 장애 원인을 빠르게 볼 수 있는 필드 정리

---

## 2) 우선순위 B (A 완료 후)

- [ ] **권한/정책 안내 UX**
  - Android/iOS 차이, NFC/위치/브라우저 제약을 사용자 눈높이로 문구 통합

- [ ] **관리자 운영 도구 보강**
  - 대량 태그 처리 시 필터/검색/내보내기
  - 실패 케이스 재처리(재요청) 동선 추가

- [ ] **실시간 알림 연계 검토**
  - 쓰기 실패/콜백 실패 임계치 초과 시 운영 알림(Slack 등) 연동

---

## 3) 기술 작업 항목

- [ ] NFC 관련 서버 액션/라우트의 에러 코드 체계 표준화
- [ ] `native-write` 관련 클라이언트 로깅 필드 정리(디바이스/경로/실패유형)
- [ ] 경로 revalidate 정책 점검(과도/누락 방지)
- [ ] E2E 테스트 케이스 보강(딥링크 진입, handoff 누락, 콜백 실패)

---

## 4) 이번 스프린트 제안(3개)

1. NFC 섹션 UI/문구 통일 (반려 상세 + 대시보드 홈)
2. `앱으로 기록` 이후 웹 피드백(대기/성공/실패) 정리
3. 태그 이력 카드의 실패 원인 노출 개선

---

## 5) 완료 기준(웹앱)

- 보호자/관리자가 “태그 쓰기 → 결과 확인 → 재시도”를 별도 설명 없이 수행 가능
- 네이티브 연동 실패 시 원인과 다음 행동이 즉시 보임
- 운영자가 로그/이력으로 장애 지점을 1분 내 파악 가능
